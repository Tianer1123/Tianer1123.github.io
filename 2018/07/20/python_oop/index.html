python面向对象和设计模式

发表于 2018-07-20 | 分类于 python | | 次阅读
字数统计: 5,698 字 | 阅读时长 ≈ 29 分钟

对象是一个数据和相关行为的集合。

  • 面向对象分析(OOA)：将问题、任务或者过程进行分析，识别其中的对象和行为的关系。产出需求。
  • 面向对象设计(OOD)：将分析结果（需求）转化为实践规范。就是设计出具体的类和接口。产出实践规范
  • 面向对象编程(OOP)：将设计结果编码成可以工作的程序的过程。产出程序

然而以上只是理论上的，实际做项目可能都是迭代式开发的。因为任何一个步骤都可能不是完美的。

面向对象分析和设计是找出对象是什么以及他们之间是如何交互的。使这些交互变得简单和直观的原则是:

  • 接口：是别的类对象可以与该对象交互的属性和行为的集合。接口需要精心设计而不能随意改变，因为接口变动，与其交互的对象都会需要发生变化。
  • 封装：将内部实现的细节隐藏起来。PYTHON中没有真正意义上的隐藏。
  • 抽象：把公共接口和私有接口分开而封装信息的过程。私有接口用于信息的隐藏。

设计模式类似于数学公式，在遇到特定问题时，如果自己没有更好的解决方案，套用这些与之对应的“数学公式”，可以帮助解决这个问题。

工厂模式

什么时候使用工厂设计模式
(a)想要追踪对象创建时。(b)想要将对象的创建和使用解耦时。(c)想要优化应用的性能和资源占用时。

建造者模式

适用的场景:
(1) 想要创建一个复杂的对象。(对象有多个部分构成，创建对象需要经过不同的步骤，这些步骤也需要一定的顺序)
(2) 一个对象有不同的表现，并且表现和创建解耦。
(3) 想要在某个时间点创建，但在稍后的某个时间访问。

在创建一个对象的时候(实例化出来的是一个空对象)，只有把对象的所有部件都创建完成，才能成为一个完整对象。例如一个网页生成器，只有把页面元素都生成完才能成为一个完整的页面。
建造者模式有两部分：建造者和指挥者。
工厂模式和创建者模式的区别类似于购买电脑，购买品牌电脑适合工厂模式，购买定制电脑适合建造者模式。

建造者中创建对象，对象中只有必要部件参数。建造者拥有建造不同表现对象的所有部件功能。指挥者告诉建造者完成那些部件。

享元模式

在我们想要优化内存使用提高应用性能之时，可以使用享元。

一个享元（Flyweight）就是一个包含状态独立的不可变（又称固有的）数据的共享对象。
若要享元模式有效，需要满足GoF的《设计模式》一书中的罗列的以下条件：

  • 应用需要使用大量的对象。
  • 对象太多，存储/渲染它们的代价太大。
  • 对象ID对于应用不重要。对象共享会造成ID比较的失败，所以不能依赖对象ID。

代理模式

在访问某个对象之前，执行一个或多个重要操作。

代理模式的四中类型:

  • 远程代理:实际存在于不同地址空间（例如，某个网络服务器）的对象在本地的代理者。
  • 虚拟代理:用于懒初始化，将一个大计算量对象的创建延迟到真正需要的时候进行。
  • 保护防护代理:控制对敏感对象的访问。
  • 智能(引用代理):在对象被访问时执行额外的动作。此类代理的例子包括引用计数和线程安全检查。

命令模式

什么情况使用命令模式？
在程序运行时，需要按照用户的意愿执行任何操作。这种情况适合使用命令行设计模式。

将“一个或一些”操作封装成一个对象，这个对象里包含这个操作的所有逻辑和方法，用户不需要知道命令执行细节，只是在需要的时候使用就可以。

命令设计模式可以提供很多好处:

  • 可以在任何时候执行一个命令，而不是在命令创建的时候。
  • 执行命令的客户端无需知道任何内部细节。
  • 可以对命令进行分组，按照一定顺序执行。

解释器模式

解释器模式用于为高级用户和领域专家提供一个类编程的框架，但没有暴露出编程语言那样的复杂性。

观察者模式

观察者模式描述单个对象（发布者，又称为主持者或可观察者）与一个或多个对象（订阅者，又称为观察者）之间的发布—订阅关系。事件驱动系统是另一个可以使用（通常也会使用）观察者模式的例子。

状态模式

状态机是一个抽象机器，有两个关键部分：状态和转换。

  • 状态:系统当前的（激活）状况。
  • 转换:指从一个状态切换到另一个状态,因某个事件或条件的触发而开始。
